# hands-control — Hand Control for Figma

Control Figma with hand gestures via webcam. Three components talk through a WebSocket relay.

## Architecture

```
tracker (browser) ──WS──► server (relay) ──WS──► figma-plugin/ui ──postMessage──► figma-plugin/code ──► Figma API
```

- **tracker** (`src/tracker.ts`, root) — webcam + MediaPipe hand landmarker, sends `hand_update` messages to relay
- **server/** — dumb WebSocket relay, groups clients into rooms, broadcasts between them
- **figma-plugin/** — Figma plugin; `ui.ts` receives hand data, `code.ts` executes Figma commands

Room IDs are generated by the plugin (e.g. `ABC-XYZ`). Tracker enters the same code to join.

## Components

### Tracker (`src/tracker.ts`, root)
- Deployed to GitHub Pages at `fab96io.github.io/hands-control`
- Build: `npm run build` (esbuild bundles `src/tracker.ts` → `dist/tracker.js`)
- Deploy: build → `git add src/ dist/` → commit → `git push origin main`

### `server/`
- Deployed to Fly.dev: `hand-relay.fly.dev`, app `hand-relay`, region `ams`
- Build happens inside Docker on deploy (no local build needed)
- Deploy: `cd server && fly deploy`
- Rooms are in-memory Maps; no persistence

### `figma-plugin/`
- Local Figma plugin only — no git remote, no deploy
- Figma loads from `figma-plugin/dist/code.js` + `figma-plugin/dist/ui.html` via `figma-plugin/manifest.json`
- Build: `cd figma-plugin && npm run build`
- After build: reload plugin in Figma
- Allowed network: `wss://hand-relay.fly.dev` (+ `ws://localhost:8080` in dev)

## Key message types

| Message | Direction | Meaning |
|---|---|---|
| `hand_update` | tracker → plugin | landmarks + gesture state, throttled at 30fps |
| `room_closed` | plugin → tracker | plugin disconnected, tracker should stop |
| `tracker_disconnected` | server → plugin | tracker left room; plugin burns room and generates new ID |

## Gestures (`figma-plugin/src/ui.ts`)

| Gesture | Action |
|---|---|
| Pinch | Drag node |
| Fist | Deselect all |
| Peace ✌️ | Undo |
| Open palm | Select node under cursor |
| 3 fingers | Pan viewport |
| Two hands | Zoom + rotate |

## Room lifecycle

1. Plugin generates `roomId`, connects to relay, displays code
2. Tracker user enters code, connects to relay
3. Data flows tracker → relay → plugin
4. On plugin disconnect: sends `room_closed` to tracker; tracker stops
5. On tracker disconnect: relay sends `tracker_disconnected` to plugin; plugin burns room, generates new ID, reconnects

## Dev setup

```bash
# Server local dev
cd server && npm run dev        # ts-node-dev on ws://localhost:8080

# Tracker local dev
npm run dev                     # esbuild watch
npm run serve                   # serve on localhost:3000

# Plugin dev
cd figma-plugin && npm run watch
```
